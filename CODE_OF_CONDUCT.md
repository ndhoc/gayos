# Code of Conduct

## Our Standard

Participation in the `gayos` project should be respectful, constructive, and technically focused.

Contributors are expected to:

- communicate respectfully
- accept technical feedback professionally
- discuss architecture and implementation decisions in good faith
- focus criticism on code, process, and design rather than people
- help keep the project welcoming to new contributors

## Unacceptable Behavior

The following behavior is not acceptable:

- harassment, insults, or personal attacks
- discriminatory language or exclusionary behavior
- bad-faith arguments, trolling, or deliberately disruptive conduct
- publishing private information without permission
- repeated hostility during technical review or project discussion

## Maintainer Responsibilities

Maintainers are responsible for:

- setting clear expectations for project collaboration
- enforcing this code of conduct fairly
- prioritizing the health of the project and its contributors
- de-escalating conflict when possible

## Enforcement

Project maintainers may remove comments, reject contributions, lock threads, or ban participation when behavior violates this code of conduct.

Actions should be proportionate to the severity and repetition of the behavior.

## Reporting

Until a dedicated contact channel is published, reports should be made through the repository issue tracker or maintainer contact listed in project metadata.
